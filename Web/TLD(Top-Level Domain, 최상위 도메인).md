✅ 1. TLD란 무엇인가?

TLD(Top-Level Domain) = 도메인의 가장 마지막 부분
예:

google.com → .com

naver.co.kr → .kr

daydot.me → .me

인터넷 도메인 구조에서 가장 상위 계층에 해당하는 구분자.

✅ 2. TLD의 주요 종류
✅ 1) gTLD (generic TLD) — 일반 최상위 도메인

가장 많이 쓰이는 일반 도메인들.

.com

.net

.org

.info

.xyz

.app, .dev (Google 관리, 자동 HTTPS)

👉 상업, 기술, 커뮤니티 등 목적 제한 X
현실에서는 “의미보다는 브랜드 느낌”이 중요.

✅ 2) ccTLD (country code TLD) — 국가 도메인

2글자로 된 국가/지역 코드 기반.

.kr (한국)

.jp (일본)

.us (미국)

.io (인도양: British Indian Ocean Territory) → 스타트업이 많이 사용해서 사실상 gTLD처럼 쓰임

👉 특정 국가 기반 서비스나 로컬 서비스에서 사용.

✅ 3) sTLD (sponsored TLD) — 특정 기관/용도

특정 조직이 관리하는 목적 기반 TLD.

.gov (미국 정부)

.edu (교육 기관)

.mil (군사)

.museum

👉 사용하려면 보통 자격 증명이 필요.

✅ 3. 왜 TLD가 중요할까?
✅ 1) 브랜딩과 신뢰도

.com은 전 세계적으로 가장 익숙하고 신뢰도가 높음.

.io, .dev, .app은 기술/스타트업 느낌을 줌.

.me는 개인 중심 서비스/블로그와 잘 맞음.

✅ 2) SEO(검색 최적화)에 영향 있음

구글은 TLD 자체만으로 유불리를 크게 주진 않지만,

**국가 코드(ccTLD)**는 로컬 검색에 영향을 준다
예: .kr은 한국 검색에 유리.

✅ 3) 도메인 가용성(availability)

.com은 유명해서 대부분 이미 선점됨.

그래서 .io, .co, .me가 많이 사용됨.

✅ 4. 실제로 많이 쓰이는 TLD TOP 5
✅ 1) .com — 전 세계 1위, 기업/서비스 표준
✅ 2) .net — .com 대안, 기술 서비스
✅ 3) .org — 비영리/커뮤니티
✅ 4) .io — 스타트업/기술 서비스 인기
✅ 5) .app / .dev — 개발자/앱 서비스용 (구글 운영, HTTPS 자동)
