오늘은 TypeScript 프로젝트의 **컴파일 및 타입 검사 규칙**을 정의하는 핵심 파일인 **`tsconfig.json`에 대해 배웠다. 이 파일은 TypeScript 컴파일러(`tsc`**)에게 소스 코드를 최종 JavaScript로 변환할 때 필요한 모든 지침을 제공하는 **프로젝트의 설계도**와 같다.

**🔑 `tsconfig.json`의 정의 및 역할**
**구분내용정의**TypeScript 컴파일러(**`tsc`**)의 동작을 제어하는 **JSON 형식의 설정 파일**.**핵심 역할**프로젝트의 **루트 파일, 컴파일 옵션, 타입 검사 규칙** 등을 지정하여 TS 코드를 JS로 변환하는 방식을 정의한다.

**📝 주요 설정 옵션 그룹**

`tsconfig.json`은 주로 세 가지 주요 기능을 통해 프로젝트의 설정을 관리한다.

**1. 컴파일 옵션 정의 (`compilerOptions`)**

**최종 JavaScript 출력물**의 형태를 결정하는 가장 중요한 섹션이다.
• **`target`**: 컴파일된 **JS 코드의 목표 ECMAScript 버전** (예: `"es2020"`).
• **`module`**: 사용할 **모듈 시스템** (예: `"commonjs"`, `"esnext"`).
• **`outDir`**: 컴파일된 JS 파일이 저장될 **출력 디렉토리**.
• **`rootDir`**: TypeScript 소스 파일의 **루트 디렉토리** 지정.

**2. 타입 검사 및 엄격성 설정**

TypeScript의 핵심 기능인 **타입 안전성**의 정도를 설정한다.
• **`strict`**: `true`로 설정하면 모든 엄격한 타입 검사 옵션을 **일괄 활성화**하여 타입스크립트의 장점을 최대한 활용한다.
• **`skipLibCheck`**: **외부 라이브러리(`.d.ts` 파일)**에 대한 타입 검사를 건너뛰어 **컴파일 속도를 향상**시킨다.

**3. 파일 포함/제외 관리 (`include`, `exclude`)**

컴파일러가 실제로 처리할 대상 파일을 지정한다.
• **`include`**: 컴파일 대상에 **포함**할 파일이나 디렉토리 경로 패턴 (예: `"src/**/*"`).
• **`exclude`**: 컴파일 대상에서 **제외**할 파일이나 디렉토리 (예: `"node_modules"`, `"dist"`).

**⚙️ 작동 방식 (컴파일 과정)**

1. 개발자가 터미널에서 **`tsc`** 명령어를 실행한다.
2. `tsc`는 프로젝트 루트에서 **`tsconfig.json`** 파일을 찾아서 읽는다.
3. 파일에 정의된 **`include`** 목록의 소스 파일을 가져온다.
4. **`compilerOptions`**에 설정된 규칙에 따라 **타입 검사**를 수행하고 최종 **JavaScript 코드로 변환**한다.
